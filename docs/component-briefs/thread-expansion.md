# Brief — Thread Expansion

- **Problem:** a deeply nested discussion (forum branch, code-review thread, design critique, support escalation) is hard to *navigate*. Users lose track of where they are, which branch holds unread replies, how to get back to the parent, and which path is selected. Rendering every nested reply is also a performance trap.
- **Use case:** structural navigation + expansion of a threaded conversation — distinct from Comment Thread, which owns comment creation / reactions / optimistic send. This component is orientation + traversal only.
- **Node data:** `id`, `parentId?` (or nested `children`), `author`, `body?` (render slot), `timestamp`, `replyCount?` (total; may exceed loaded children → lazy load), `unreadCount?` (branch total), `unread?` (this node), `resolved?`, `deleted?`, `collapsed?`, `metadata?`.
- **States:** collapsed / expanded branch · selected (deep-linked) · in-selected-path · unread node · unread-in-branch badge · resolved (optionally collapsed) · deleted placeholder · loading replies · failed reply load · empty branch.
- **Main interaction:** expand/collapse branch, roving-tabindex tree keyboard nav (↑/↓ move, →/← expand-into / collapse-to-parent, Home/End, Enter/Space select), **Next unread** (walks unread branches, expanding the path + preserving focus), **Go to parent**, **Expand all** (safe depth), **Collapse all**, selected-path breadcrumb (click to jump), lazy **Load more** replies with loading/error/retry rows.
- **Structure safety:** indentation is capped (`MAX_INDENT_DEPTH`) and auto-expand / expand-all never exceed `maxAutoDepth`, so a pathological deep thread never renders unbounded rows. Collapsed branches aren't rendered.
- **Animation purpose:** short per-row fade on expand/collapse only — **no large stagger** for deep conversations; chevron rotation. Reduced motion → instant.
- **API sketch:** `nodes`, `rootId?`, `selectedId?`/`defaultSelectedId?`/`onSelect?`, `expandedIds?`/`defaultExpandedIds?`/`onExpandedChange?`, `defaultExpandDepth?`, `onLoadMore?`, `loadingId?`, `errorId?`/`errorMessage?`/`onRetryLoad?`, `onNavigateParent?`, `onNavigateUnread?`, `collapseResolved?`, `maxAutoDepth?`, `formatTimestamp?`, `label?`, `maxHeight?`.
- **Accessibility:** `role="tree"` / `treeitem` with `aria-level`, `aria-expanded`, `aria-selected`, roving tabindex; parent/child conveyed by structure + ArrowLeft-to-parent; focus preserved across jumps and when a branch containing focus collapses; selected path rendered as text (breadcrumb); unread conveyed by **text** ("Unread" / "N unread in branch"), not colour alone; loading (`aria-busy`) + error rows labelled; polite live region announces navigation; reduced-motion.
- **Mobile:** indentation capped so deep branches don't push off-screen; toolbar wraps; single column.
- **Dependencies:** motion + `@motionstack/utils` (cn) + `@motionstack/primitives` (`useReducedMotion`, `formatTimestamp`). No new deps. Presentation only — app owns data, unread accounting, resolution, lazy loading.
- **Similarity concern:** tree/thread navigators are common; differentiate via unread-branch traversal + selected-path breadcrumb + focus preservation + safe-depth structural limit. Low–moderate.
- **Tier:** Free.
- **Release criteria:** rapid gate + expand/collapse, next-unread navigation, focus preservation, deleted placeholder, and loading/error branch states verified.
