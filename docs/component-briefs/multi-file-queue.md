# Brief — Multi-file Queue

- **Tier:** Free · **Category:** File workflows
- **Problem:** Apps that import/transfer *many* files at once need a queue *manager* — priority scheduling, bounded concurrency, dependency blocking, and queue-wide operations — not just one progress bar. This is rebuilt ad hoc and usually inaccessibly.
- **Use case:** A batch media-import / transfer surface the app drives. The component presents the app-supplied collection grouped into priority lanes, visualises how many concurrency slots are occupied, explains blocked items, and emits intent callbacks. It never imports/uploads or advances progress itself.
- **Main states:** empty · offline · running · all-paused · partial failure. Per-item: `queued · waiting · active · paused · blocked · processing · completed · failed · cancelled` (icon + text, never colour alone). Priority: `high · normal · low` (labelled badge + lane, never colour alone).
- **Main interaction:** per-item reorder (within lane), raise priority, pause/resume/retry/cancel/remove; queue-level pause-all / resume-all / retry-failed / clear-completed; add files.
- **Animation purpose:** communicate queue changes — insert, reorder (`layout="position"`), queued→active, pause, complete, fail, removal — not decoration. No constant bouncing, no per-frame React state, no per-% announcements, no replaying every item when one changes. Reduced motion renders final state, bars snap.
- **API sketch:** `<MultiFileQueue items concurrency onAdd onRemove onPause onResume onRetry onCancel onReorder onPriorityChange onPauseAll onResumeAll onRetryFailed onClearCompleted offline? compact? renderItemDetails? />`; item = `{ id, fileName, fileType, fileSize, priority, queuePosition?, status, progress, speed?, remainingTime?, retryCount?, error?, thumbnail?, dependency?, destination?, metadata? }`.
- **Accessibility:** semantic list per lane; `role=progressbar` per item + queue-level; status/priority in text; blocked + error text associated via `aria-describedby`; reorder and priority are ordinary keyboard buttons that announce the new lane position in a polite live region; announcements throttled to status transitions + moves (never every %); focus preserved onto a neighbour after removal; no colour-only status/priority; reduced-motion safe.
- **Mobile behavior:** stacked rows, large touch targets, lanes reflow vertically, optional `compact` density; controls wrap.
- **Dependencies:** `motion` + `@motionkit/{utils,primitives}` (`useReducedMotion`, `statusVars`). No Node/`next`/Remotion imports.
- **Similarity concern:** distinct from File Upload Pipeline (one upload workflow) — this manages a *collection* with priority lanes, concurrency slots, dependency blocking, and queue-level operations. Generic queue pattern; clean-room implementation, original API + composition. Low.
- **Release criteria:** ~5 targeted tests (keyboard reorder + position announcement, focus-after-removal, pause-all/resume-all state, priority raise) + axe; rapid gate (desktop/mobile/light/dark/reduced/keyboard, typecheck, build, registry validates, clean-fixture install, no forbidden imports, no console/hydration errors).
